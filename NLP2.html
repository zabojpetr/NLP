2. Cross-Entropie a modelování jazyka¶

Rozdělení na datasety:¶

  • poslednch 20 000 testovac data
  • poslednch 40 000 ze zbytku heldout data
  • zbytek trnovac data

Spočítat pravděpodobnosti¶

$T$ velikost textu, $V$ velikost slovnku
$ p_0(w_i) = \dfrac{1}{V}\\ p_1(w_i) = \dfrac{c_1(w_i)}{T}\\ p_2(w_i|w_{i-1}) = \dfrac{c_2(w_{i-1},w_i)}{c_1({w_{i-1})}}\\ p_3(w_i|w_{i-2},w_{i-1}) = \dfrac{c_3(w_{i-2},w_{i-1},w_i)}{c_2(w_{i-1},w_i)} $

EM algoritmus¶

Vypotat parametry z heldout dat (tak z trnovacch a pot zahodit) $$p_s(w_i|w_{i-2},w_{i-1}) = l_0p_0(w_i) + l_1p_1(w_i) + l_2p_2(w_i|w_{i-1})+l_3p_3(w_i|w_{i-2},w_{i-1})$$ pro $l_0+l_1+l_2+l_3 = 1$

Vypočítat cross-entropii¶

Poupravit parametry vyhlazen nsledujcm zpsobem: pidat $10\%$, $20\%$, $\ldots$,$90\%$, $95\%$ a $99\%$ rozdlu mezi trigram vyhlazen a $1.0$ k jeho hodnot, proporciln uprav zbvajc ti parametry (pipomeme, e maj souet 1!!). Potom nastavte trigram vyhlazovac parametr na $90\%$, $80\%$, $\ldots$, $10\%$, $0\%$ z jeho hodnoty a proporciln uprav zbvajc parametry. def create_datasets(data):
    test_data = data[-TESTDATASIZE:]
    heldout_data = data[-(TESTDATASIZE+HELDOUTDATASIZE):-TESTDATASIZE]
    train_data = data[:-(TESTDATASIZE+HELDOUTDATASIZE)]
        
    return (train_data,heldout_data,test_data)

def create_counts(data):
    # Odest 1 od etnosti poslednho slova, protoe to u nem dvojici a j chci z toho potat psti
    words, counts = np.unique(data, return_counts=True)
    counts_words = dict(zip(words,counts))
    
    counts_words[data[-1]] -= 1
    
    counts_pairs = dict()
    for i in range(data.size - 1):
        if (data[i],data[i+1]) in counts_pairs:
            counts_pairs[(data[i],data[i+1])] += 1
        else:
            counts_pairs[(data[i],data[i+1])] = 1
    
    counts_triples = dict()
    for i in range(data.size - 2):
        if (data[i],data[i+1],data[i+2]) in counts_triples:
            counts_triples[(data[i],data[i+1],data[i+2])] += 1
        else:
            counts_triples[(data[i],data[i+1],data[i+2])] = 1
        
    return (counts_words,counts_pairs,counts_triples)

def p0():
    return 1/len(COUNTS_WORDS)

def p1(w1):
    if w1 not in COUNTS_WORDS:
        return 0
    return COUNTS_WORDS[w1]/T

def p2(w1,w2):
    if w1 not in COUNTS_WORDS or COUNTS_WORDS[w1]==0:
        return p0()
    if not (w1,w2) in COUNTS_PAIRS:
        return 0
    return COUNTS_PAIRS[(w1,w2)]/COUNTS_WORDS[w1]

def p3(w1,w2,w3):
    if (w1,w2) not in COUNTS_PAIRS or COUNTS_PAIRS[(w1,w2)] == 0:
        return p0()
    if not (w1,w2,w3) in COUNTS_TRIPLES:
        return 0
    return COUNTS_TRIPLES[(w1,w2,w3)]/COUNTS_PAIRS[(w1,w2)]

def p_lambda(w1,w2,w3,lambdas):
    return lambdas[3]*p3(w1,w2,w3) \
          +lambdas[2]*p2(w2,w3) \
          +lambdas[1]*p1(w3) \
          +lambdas[0]*p0()

def EM(epsilon, heldout_data):
    lambdas = [0.25]*4
    history = [lambdas]
    
    last_value = 0
    with tqdm_notebook(total=1000,desc="EM", leave=False) as pbar:
    
        while(True):
            expected_counts = [0]*4

            for i in range(heldout_data.size-2):
                p_lam = p_lambda(heldout_data[i],heldout_data[i+1],heldout_data[i+2],lambdas)
                if(p_lam==0):
                    continue
                expected_counts[0] += lambdas[0]*p0()\
                                     /p_lam
                expected_counts[1] += lambdas[1]*p1(heldout_data[i+2])\
                                     /p_lam
                expected_counts[2] += lambdas[2]*p2(heldout_data[i+1],heldout_data[i+2])\
                                     /p_lam
                expected_counts[3] += lambdas[3]*p3(heldout_data[i],heldout_data[i+1],heldout_data[i+2])\
                                     /p_lam

            new_lambdas = [x/sum(expected_counts) for x in expected_counts]

            is_end = True
            maximum = 0
            for i in range(len(lambdas)):
                if(abs(lambdas[i]-new_lambdas[i]) > maximum):
                    maximum = abs(lambdas[i]-new_lambdas[i])
                if(abs(lambdas[i]-new_lambdas[i])>epsilon):
                    is_end = False
                    break

            if is_end:
                pbar.update(1000-pbar.last_print_n)
                break

            lambdas = new_lambdas
            history.append(lambdas)
            
            #Progress bar
            new_value = 1/math.log(EPSILON,10)*math.log(maximum,10)
            pbar.update(int((new_value - last_value)*1000) if new_value - last_value > 0 else 0)
            last_value = new_value if new_value > last_value else last_value
    
        
    
    return (lambdas,history)

def entropy(data, lambdas):
    H = 0
    for i in range(data.size-2):
        H -= math.log2(p_lambda(data[i],data[i+1],data[i+2],lambdas))
        
    return H/(data.size-2)

Pomocn metody k zmnn parametr o urit procenta. Vypote se nov $\lambda_3$ a pot se pomrov zbytek hodnoty piad ostatnm parametrm.

def lambda_increase(lambdas, perc):
    lambdas[-1] += (1 - lambdas[-1])*perc
    ratio = (sum(lambdas)-lambdas[-1])/(1-lambdas[-1])
    lambdas = [x/ratio for x in lambdas]
    lambdas[-1] *= ratio
    return lambdas

def lambda_decrease(lambdas, perc):
    lambdas[-1] *= perc
    ratio = (sum(lambdas)-lambdas[-1])/(1-lambdas[-1])
    lambdas = [x/ratio for x in lambdas]
    lambdas[-1] *= ratio
    return lambdas

def coverage(test_data):
    test_counts = create_counts(test_data)
    train_counts = (COUNTS_WORDS,COUNTS_PAIRS,COUNTS_TRIPLES)
    cov = [0]*3
    for i in range(len(cov)):
        for t in test_counts[i]:
            if t in train_counts[i]:
                cov[i] += test_counts[i][t]
        cov[i] /= sum([test_counts[i][x] for x in test_counts[i]])
    return cov

Nastavení parametrů při výpočtu na trénovacích datech¶

Pi bhu EM algoritmu na trnovacch datech dostaneme 1 u parametru $\lambda_3$ a u ostatnch 0 (na velikosti $\epsilon$ zle kolik pedchozch parametr se pln vynuluje). To je zpsobeno tm, e pro vechny trojice dokeme spotat pravdpodobnost jejich vskytu a proto, e podmnn pravdpodobnost trojice (2 slova z historie a jedno aktuln) je vt rovna podmnn pravdpodobnosti dvojice (pouze jedno slovo z historie). A protoe EM algoritmus maximaluzuje oekvanou hodnotu, mus tedy vynulovat vechny parametry krom $\lambda_3$.

Zvýšení parametru $\lambda_3$¶

Pi zvyovn parametru $\lambda_3$ rychle roste cross-entropie. To je dno tm, e dky zvten $\lambda_3$ mme pocit, e s vy pravdpodobnost dokeme odhadnou trojici, ale protoe jsme parametr zvtili umle, tato pedstava je miln a nemme nic, m bychom to mohli napravit. Tedy m vce zvtujeme $\lambda_3$, tm astji mme chybn odhad a proto se entropie zvtuje.

Snížení parametru $\lambda_3$¶

Zde cross-entropie tak roste, ale ji ne tak vrazn. Je to zpsobeno tm, e snen parametru $\lambda_3$ nm me jet "zachrnit" parametr $\lambda_2$, sice bychom doshli lepho vsledku s del histori, ale dvojice jsou podmnoiny trojic, tedy nedojde k takovmu zhoren jako v opanm ppad, kdy chceme vechno odhadovat podle trojic.
